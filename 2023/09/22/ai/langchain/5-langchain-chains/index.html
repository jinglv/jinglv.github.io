LangChain核心模块chains（早期）

发表于 2023-09-22 分类于 LangChain
学习LangChain学习笔记第四讲

简介

Chain 作为 LangChain 的核心模块之一，重要性不言而喻，它相当于是所有复杂逻辑的基础，用来将每个零散的逻辑串联成一整个业务流程，Chain 的设计非常巧妙，可以说是大模型应用的最佳实践之一。

Chain的设计非常巧妙，也非常值得借鉴，也是为什么LangChain能火爆的主要原因之一。

设计思路

Chain 的设计非常巧妙，也非常值得借鉴，也是为什么 LangChain 能火爆的主要原因之一。

Chain 的一个主要作用，就是根据需求，可以将各种能力拼接整合。比如上图，LangChain 内部包含多个模块，可以定制 LLMChain 只使用 Prompt 和 LLM 模块，也可以定制一个CustomChain（自定义Chain） 使用了 LLM、Parser、Agent模块。

Chains主要模块

Chains主要包含以下几个模块：

模块 应用场景
LLMChain 基础链
SimpleSequentialChain 单输入/输出串联调用
SequentialChain 更通用形式的顺序链，允许多个输入/输出
LLMRouteChain 实现条件判断的链

LLMRouteChain

LLMRouteChain 的主要作用是能根据提示词的不同而选择不同的Chain进行执行（类似于Python代码中的分支判断的作用）。

实现步骤如下：

  1. 加载 Token，创建大模型实例
  2. 创建 destination_chains，目的是构造各个分支，以及其对应的prompt
  3. 通过模板和大模型对象，生成LLMRouterChain，用于实现分支逻辑。而其能实现分支逻辑的原理，也是通过 prompt 实现，注意示例中的变量MULTI_PROMPT_ROUTER_TEMPLATE对应的内容以及信息
  4. 创建一个默认的LLMChain实例，作为上述匹配未命中时的默认调用目标，避免调用最终没有逻辑去处理的情况出现
  5. 通过 MultiPromptChain 将以上三个模块整合，最终实现一个支持分支判断的 Chain
